# AGENTS.md

## Repository purpose
This repository is a curated awesome list for digital assets in financial services, with a strong European and institutional perspective.

## Core objective
Help maintain and improve the repository with conservative, high-quality changes.
Prioritize curation quality over quantity.

## Editing guidelines
- Preserve the existing tone, structure, and editorial style of the README.
- Prefer institutional, regulatory, research-heavy, and ecosystem-relevant sources.
- Prioritize Europe, but include globally relevant sources when they materially affect European markets.
- Avoid low-substance vendor marketing pages unless they are strategically important market infrastructure announcements.
- Do not remove entries unless they are dead links, duplicates, clearly outdated, or not relevant enough for this list.
- Keep descriptions concise, neutral, and informative.
- Do not invent facts, dates, or summaries.
- If unsure, add items to `candidates.md` instead of editing the main README aggressively.

## Review guidelines
- Flag dead links.
- Flag duplicate coverage.
- Flag categories with weak or outdated coverage.
- Suggest wording consistency improvements.
- Prefer draft pull requests over direct finalization.

## Language and style quality
- Ensure all entries are free of spelling and grammar errors.
- Maintain a consistent, professional, and neutral tone.
- Use concise one-sentence descriptions where possible.
- Avoid overly promotional or subjective wording.
- Ensure consistent terminology, especially for terms such as:
  - DLT
  - tokenization
  - digital assets
  - stablecoins
  - CBDC
- Normalize capitalization, punctuation, spacing, and markdown formatting.
- Prefer clear and precise phrasing over verbose wording.

## Style consistency rules
Descriptions should ideally follow this pattern:

[What it is] + [why it matters]

Example:
- A BIS report analyzing the role of tokenized deposits in future financial market infrastructures.

Avoid:
- vague phrases such as "interesting" or "important"
- marketing language such as "leading solution" or "cutting-edge"
- inconsistent tense usage

## Safety rails
- Do not change the meaning of existing entries.
- Do not introduce new claims or interpretations.
- If a description is unclear, flag it instead of rewriting aggressively.
- When confidence is low, prefer `candidates.md` or PR notes over direct README edits.

## Output format
When proposing changes:
1. Add a short summary.
2. Separate suggestions into:
   - Add
   - Review
   - Remove/Fix
   - Language Improvements
3. Explain why each proposed source is relevant.
4. Keep changes conservative unless explicitly asked to do a broader cleanup.
